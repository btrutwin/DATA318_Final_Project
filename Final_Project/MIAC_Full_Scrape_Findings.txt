MIAC Baseball 2026 Web Scraper - Exploration Findings
======================================================
Date: April 17, 2026
Author: Generated by Copilot during scraper development


1. CALENDAR / COMPOSITE SCHEDULE PAGINATION
============================================

KEY FINDING: The MIAC composite schedule is NOT paginated by month.

I tested the following URLs:
  - https://miacathletics.com/calendar.aspx?path=baseball
  - https://miacathletics.com/calendar.aspx?path=baseball&year=2026&month=2
  - https://miacathletics.com/calendar.aspx?path=baseball&year=2026&month=3
  - https://miacathletics.com/calendar.aspx?path=baseball&year=2026&month=4

ALL of these URLs return the EXACT SAME full-season schedule. The "month"
parameter in the URL does not filter the displayed games. Every request
returns the complete schedule from February 14, 2026 through May 5, 2026.

This means:
  - A single request to the base URL is sufficient to get all games.
  - The script loops through months 1-12 as a safety net, then deduplicates.
  - If the MIAC ever changes to true month-based pagination, the loop
    approach will still work correctly.

Season date range: Saturday 2/14/2026 through Tuesday 5/5/2026.


2. CALENDAR TABLE STRUCTURE
============================

Each game row in the HTML table has this format:

  | Date Cell | Away Team + Score | Home Team + Score | Result | Location | Box Score Link |

Examples:
  | Saturday 2/14/2026 | Hamline 4 | Buena Vista University 0 | Final 9 innings | Lincoln, NE // Sandhills Global Youth Complex | Box Score |
  | Sunday 3/15/2026 | Saint Mary's | Millikin University | Cancelled | Decatur, IL // Workman Family Baseball Field |   |

Key observations:
  - Team name and score are combined in the same cell (e.g., "Hamline 4").
  - For cancelled/postponed games, no score is present (just the team name).
  - The result column contains values like:
      "Final"
      "Final 9 innings"
      "Final (7 Innings)"
      "Final (8 inn.)"
      "Final (10 inn.)"
      "Final (12 Innings)"
      "Final 7 innings"
      "Final 10 innings"
      "Final 11 innings"
      "Cancelled"
      "Postponed"
  - Box Score link is only present for completed ("Final") games.
  - Conference games are identified by "(Conf.)" appearing in the location.
  - Double headers are identified by "Double Header" in the location.
  - Rows between date groups are empty separator rows.


3. BOX SCORE PAGE STRUCTURE
============================

Each box score page (e.g., linked from the calendar) uses the Sidearm Sports
platform and contains:

  a) LINESCORE TABLE: Contains an "R" (runs) column.
     - Row 1 = away team, Row 2 = home team.
     - Also has columns for each inning plus H (hits) and E (errors).

  b) PITCHING TABLES: Tables with an "IP" (innings pitched) column.
     - Typically two pitching tables per box score: away team first, home team second.
     - First row of each pitching table = starting pitcher.
     - Player column may be labeled "Player", "Name", or similar.
     - Last row may be a "Totals" summary row (must be filtered out).

  c) BATTING TABLES: Present but not scraped in this project.

The script identifies tables by column patterns (looking for "R", "IP", etc.)
rather than relying on fixed table indices, which makes it robust to minor
HTML layout changes.


4. STATS PAGE STRUCTURE
========================

URL: https://miacathletics.com/stats.aspx?path=baseball&year=2026

The stats page contains multiple HTML tables. The page has tabs/sections for:
  - OVERALL (default view)
  - INDIVIDUAL
  - LEADERS
  - RESULTS
  - GAME HIGHS

The "Overall" section contains these tables (with identifying column patterns):

  a) Team Batting:
     Columns: Index, Team, AVG, G, AB, R, H, 2B, 3B, HR, RBI, TB, SLG%, BB,
              HBP, SO, GDP, OB%, SF, SH, SB-ATT, PO, A, E, FLD%

  b) Team Pitching:
     Columns: Index, Team, ERA, W-L, G, SHO, SV, IP, H, R, ER, BB, SO,
              2B, 3B, HR, AB/B, AVG, WP, HBP, BK, SFA, SHA

  c) Team Fielding:
     Columns: Index, Team, C, PO, A, E, FLD%, DP, SBA, CSB, SBA%, PB, CI

The script identifies tables by their column signatures rather than fixed
table indices (which was the approach in the original MIAC_Scraper.Rmd).
This is more robust because:
  - The MIAC site may add/remove tables over time.
  - The ordering of tables could change.
  - Additional tables (conference-only stats, etc.) may be present.


5. DATA QUALITY NOTES
======================

  - Some games are listed as "Cancelled" or "Postponed" -- these have no
    scores and no box score links. The scraper handles these gracefully.

  - Team names in the schedule include non-MIAC opponents (non-conference
    games), so the dataset includes all MIAC team games, not just
    conference-vs-conference matchups.

  - The "location" field often contains the venue name after a "//" separator
    (e.g., "Lincoln, NE // Sandhills Global Youth Complex").

  - The "result" field has inconsistent formatting for extra-inning games:
    sometimes "Final 10 innings" (no parens), sometimes "Final (10 inn.)",
    sometimes "Final (7 Innings)" with capital I. The scraper preserves the
    raw text and adds a cleaned "status" column.

  - As of April 17, 2026, the schedule shows games through at least
    April 15. Games scheduled through early May may still be upcoming.


6. OUTPUT FILES
================

The scraper produces these 8 CSV files:

  1. team_batting_2026.csv        - Team cumulative batting stats
  2. team_pitching_2026.csv       - Team cumulative pitching stats
  3. team_fielding_2026.csv       - Team cumulative fielding stats
  4. individual_batting_2026.csv  - Individual player batting stats
  5. individual_pitching_2026.csv - Individual player pitching stats
  6. miac_games_2026.csv          - Cleaned game schedule (no box score data)
  7. miac_boxscores_2026.csv      - Box score data only (runs, starters, IP)
  8. miac_games_full_2026.csv     - Games merged with box score data

PITCHING FEATURE ENGINEERING SUMMARY 
====================================================

Rolling ERA Proxy (home_era_proxy_last5, away_era_proxy_last5):
- Computed from game-level data as (runs allowed × 9) / starter IP, averaged over the last 5 games for each team.
- Uses only the starter's innings pitched and total runs allowed (not earned runs, not full team IP).
- This is a proxy, not a true team ERA. It is the only rolling pitching stat possible with available game-level data.

Season Team Pitching Stats (home_era_season, etc.):
- ERA: Directly from the ERA column in team_pitching_2026.csv (season total, not rolling).
- WHIP: (BB + H) / IP, using season totals from team_pitching_2026.csv.
- K/9: (SO × 9) / IP, using season totals.
- BB/9: (BB × 9) / IP, using season totals.
- Opponent BA: H / AB, using season totals.

What was NOT used (and why):
- True team ERA at the game level: Not possible because game-level data does not include team total IP or team earned runs for each game.
- Rolling team WHIP, K/9, BB/9, OppBA: Not possible at the game level for the same reason—no per-game team H, BB, SO, or AB.
